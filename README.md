

# API
Website voor het vak API 

<h2>Dag 1 (01-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>
<p>Vandaag ben ik begonnen met het maken van een idee voor het project. Ik wil graag iets doen wat te maken heeft met anime. Ik dacht aan een website waar mensen een favoriete anime kunnen vinden en zien hoe de characters het best met hun koppelen in de zin van emoties en karakter. Ik ben nog aan het uitvogelen welke web apis ik ga gebruiken maar qua content api heb ik al Jikan. Dit is de grootste anime api die beschikbaar is. </p>

<h2>Dag 2 (02-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>
<p>Ik heb mijn idee gepitches aan Jad en Cyd en ik ben erachter gekomen dat het idee veelte simpel is. Ik moest wat anders bedenken dus ik ben verder gaan brainstormen. Ik ben een jaar terug begonnen met schaken dus ik wou misschien daar iets mee doen. Ik kwam op het idee van een schaakwebsite waar gebruikers hun partijen kunnen opzoeken en dan op een mooie scherm kunnen zien wat hun statestieken zijn etc. Ik heb dit idee weer gepitched aan Jad en hij vondt dit wat beter. Ik moest wel nog nadenken over welke API's ik ging gebruiken. </p>

<h2>Dag 3 (08-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>
<p>Ik heb vandaag nagedacht over welke api's ik ging gebruiken. Ik ben gekomen op deze api's: Chess.com
users + games

Lichess
analyse

Stockfish
accuracy / best move

Chart.js
grafieken 

Ik heb dit weer gepitched aan Jad en hij vond dat chart.js iets te makkelijk was hoe ik het wou gebruiken en kwam met de suggestie om view transition te gebruiken. Ik ben dus uiteindelijk daar voor gegaan + nog een andere web api. Maar ik kan gelukkig nu eindelijk een begin maken. </p>

<h2>Dag 4 (09-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>
<p>Ik ben begonnen met het maken van de index scherm. Ik heb de chess.com api gebruikt om users op te zoeken en begin van stats zoals hun ELO. Ik ben daarna bezig geweest met hoe view transitions werkt op astro. Ik ben erachter gekomen dat het helemaal geen view transtions heet maar client router. Dit is ook alleen maar te gebruiken wanneer je meerdere pages hebt dus daar moet ik eerst aan beginnen. Ik ben aan het nadenken hoe ik de pages ga verdelen. Ik dacht aan dat de recente games worden laten zien aan de rechter kant en dat wanneer je klikt op een game dat het wordt laten zien op de volgende scherm.  </p>

<h2>Dag 5 (15-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>

Ik probeerde een header te maken als component maar dit kwam uiteindelijk niet uit als ik wilde dus ik heb geen header gemaakt. Ik heb wel een layout gemaakt die ik later wil gaan gebruiken voor de volgende pagina.  </p>

<h2>Dag 6 (16-04-2026)</h2>
<h3>Wat heb ik gedaan vandaag?</h3>
<p>Ik ben begonnen met de tweede pagina. Ik heb de wit en zwart verdeeld tussen twee kanten en je kan duidelijk zien wie de winaar is. Ik ben nog aan het uitzoeken welke stats ik precies wil laten zien maar ik heb al wel een paar dingen toegevoegd zoals de openingsnaam en de rating van beide spelers. Ik ben ook bezig geweest met het maken van een view transition maar dit is nog niet helemaal gelukt. Ik heb Jad gevraagt om hulp en hij heeft het uiteindelijk kunnen fixen. 
</p>
